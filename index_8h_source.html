115 * \section theory Theory
116 * Mutual information is used to measure how much two random variables
117 * are correlated, including both linear and non-linear relationships.
118 * Imagine we have a set of data pairs \f$(x_i, y_i)\f$, where each pair
119 * is an independent realization of random variables \f$(X, Y)\f$.
120 * These variables follow a distribution \f$\mu(x, y)\f$.
121 * Shannon entropy, denoted as \f$H(X)\f$, is calculated using:
122 * \f[
123 * H(X) = -\int\mu(x)\log\mu(x)dx
124 * \f]
125 * where the logarithm's base determines the information's unit
126 * (bits, nats, etc.). We use the natural logarithm in our context.
127 * Mutual information, \f$I(X, Y)\f$, is defined as:
128 * \f[
129 * I(X, Y) = H(X) + H(Y) - H(X, Y)
130 * \f]
131 * This value indicates how strongly \f$X\f$ and \f$Y\f$ are connected.
132 * If they are completely independent, \f$I(X, Y)\f$ equals zero.
133 * Often, we don't know \f$\mu\f$ exactly and need to estimate it.
134 * Assuming \f$\mu\f$ is uniform, we approximate \f$H(X)\f$ with:
135 * \f[
136 * \widehat{H}(X) = -\frac{1}{N}\sum_{i=1}^N\widehat{\log(\mu(x_i))}
137 * \f]
138 * We use a k-nearest neighbor estimator for this purpose.
139 * To calculate the probability distributions necessary for these
140 * estimations, we consider the distances to a data point's nearest
141 * neighbors in both X and Y dimensions, and compute probabilities based
142 * on these distances.

145 * \section Algorithms
146 *
147 * \subsection mutual_information_gpu Parallel Mutual Information
148 * | Variable | Description |
149 * |----------|-------------|
150 * | `Xa` | Array containing the data points for the first random variable in the mutual information calculation. |
151 * | `Xb` | Array containing the data points for the second random variable in the mutual information calculation. |
152 * | `k` | Integer specifying the number of nearest neighbors to consider for each data point. |
153 * | `n` | Integer representing the total number of data points in each of the random variables `Xa` and `Xb`. |
154 * | `nXa` | Array for storing the count of data points in `Xa` within a radius of epsilon_Xa / 2 for each point. |
155 * | `nXb` | Array for storing the count of data points in `Xb` within a radius of epsilon_Xb / 2 for each point. |
156 * | `s_argMin` | Shared memory array used to store the indices of the nearest neighbors during the calculation. |
157 * | `s_min` | Shared memory array used to store the minimum distances calculated in the search for nearest neighbors. |
158 * | `s_epsXa` | Shared memory array to store the maximum distance (epsilon) within `Xa` for the k-th nearest neighbors. |
159 * | `s_epsXb` | Shared memory array to store the maximum distance (epsilon) within `Xb` for the k-th nearest neighbors. |
160 * | `s_nXa` | Shared memory array used to temporarily store neighbor counts for `Xa` within each CUDA block. |
161 * | `s_nXb` | Shared memory array used to temporarily store neighbor counts for `Xb` within each CUDA block. |
162 *
163 * \subsubsection step1 Step 1: Initialization
164 * - \b Input: Arrays \c Xa, \c Xb, integers \c k, \c n, output arrays \c nXa, \c nXb
165 * - \b Output: Updated \c nXa, \c nXb with neighbor counts for each point in \c Xa, \c Xb
166 * - Initialize shared memory arrays: \c s_argMin[1024], \c s_min[1024], \c s_epsXa[1], \c s_epsXb[1], \c s_nXa[1024], \c s_nXb[1024]
167 *
168 * \subsubsection step2 Step 2: Parallel Processing
169 * For each data point \c i processed in parallel CUDA blocks:
170 * - Load \c Xa[i] and \c Xb[i] into registers \c r_Xai, \c r_Xbi
171 * - Set local thread index \c localThreadIndex = threadIdx.x
172 * - Initialize \c s_nXa[localThreadIndex], \c s_nXb[localThreadIndex] to zero
173 * - If \c localThreadIndex == 0, set \c s_epsXa[0], \c s_epsXb[0] to zero
174 *
175 * \subsubsection step3 Step 3: Load Data in Chunks
176 * - Iterate over \c Xa, \c Xb in chunks, loading into \c r_Xa, \c r_Xb
177 * - Synchronize threads using \c __syncthreads()
178 *
179 * \subsubsection step4 Step 4: Find k Nearest Neighbors
180 * - Initialize \c localMin = RAND_MAX, \c localArgMin = 0
181 * - Iterate over chunks, updating \c localMin, \c localArgMin based on distance \c dX between \c r_Xai, \c r_Xbi and chunk data
182 * - Update shared memory \c s_min, \c s_argMin
183 * - Perform parallel reduction to find global minimum distance and corresponding index
184 * - Update \c s_epsXa, \c s_epsXb and mark processed points in \c r_Xa, \c r_Xb as needed
185 * - Synchronize threads using \c __syncthreads()
186 *
187 * \subsubsection step5 Step 5: Increment Neighbor Counts
188 * - Iterate over chunks, incrementing \c s_nXa, \c s_nXb based on distance conditions to \c r_Xai, \c r_Xbi
189 * - Synchronize threads using \c __syncthreads()
190 * - Perform parallel reduction on \c s_nXa, \c s_nXb
191 *
192 * \subsubsection step6 Step 6: Update Global Counts
193 * - If \c localThreadIndex == 0, update \c nXa[i], \c nXb[i] with reduced counts from \c s_nXa[0], \c s_nXb[0]
194 * - Synchronize threads using \c __syncthreads()
